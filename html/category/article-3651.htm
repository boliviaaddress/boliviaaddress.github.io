maven项目关于target目录没有生成xml、properties等文件问题

  • 首页
  • 文章分类
  • 正文

问题描述：
我在maven第一次弄了父子项目，然后tomcat，启动失败，报异常： class path resource [applicationContext.xml] cannot be opened because it does not exist
再看target目录没有配置文件
解决方案：
需要在pom.xml文件中添加:

<build><resources><resource><directory>src/main/java</directory><!--所在目录--><includes><include>**/*.properties</include><include>**/*.xml</include></includes><filtering>false</filtering></resource><resource><directory>src/main/resources</directory><!--所在目录--><includes><include>**/*.properties</include><include>**/*.xml</include></includes><filtering>false</filtering></resource></resources></build>

还有就是我把子项目删了。。因为原来的项目本来没有打算当做父项目，然后原来的项目成为父项目之前可以运行，成为了父项目就运行不了了，然后我把子项目删除了
原来的项目要： pom改成 war或者jar
< packaging>war</ packaging> < packaging>jar</ packaging>

<groupId>com.scl</groupId><artifactId>Springmvc-Test01</artifactId><version>1.0-SNAPSHOT</version><packaging>war</packaging>
